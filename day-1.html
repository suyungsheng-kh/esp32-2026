Day 1：有聲紅綠燈 —— 交通指揮官

🎯 專題目標：有聲音的紅綠燈系統

因應前導課程接線練習，第一天我們將專注於「數位訊號 (0 與 1)」的輸入與輸出，完成基礎的燈光與聲音控制，並打造實用的交通號誌系統。

模組原理與概念

LED 燈 (數位輸出)

輸出高電位(1)亮、低電位(0)暗。ESP32 擴充板上通常有一顆內建燈可直接控制測試，外部 LED 則需注意接線正確，電流只能單向通過。

按鈕模組 (數位輸入)

按鈕就是橋樑。按下時導通電流，ESP32 會讀取到 1 (High)；放開時斷開，ESP32 讀取到 0 (Low)。這是最基本的人機互動。

蜂鳴器 (數位輸出)

通電就會發出聲音。透過程式控制它「通電-等待-斷電-等待」的頻率與間隔，就可以製造出像紅綠燈倒數的「滴、滴、滴」提示音。

模組小試身手：基礎功能測試

0
內建LED閃爍

無需額外接線

🎯 任務：寫一個迴圈測試燈號交替：

  • 測試一：延遲毫秒 1000
  • 測試二：延遲毫秒 500
  • 測試三：延遲毫秒 100
🤔 思考：觀察燈號閃爍效果有什麼不同？
1
神奇開關

接線腳位 (按鈕 ➜ LED)：

  • 綠：Pin 33 ➜ Pin 16
  • 黃：Pin 34 ➜ Pin 17
  • 紅：Pin 35 ➜ Pin 18

🎯 任務：寫 3 組「如果...否則...」。按下紅按鈕亮紅燈，綠按鈕亮綠燈，黃按鈕亮黃燈。

💡 提示：積木請使用「數位讀出腳位(上拉電阻)[#]」
2
自動紅綠燈

沿用任務 1 的 LED

🎯 任務：不按按鈕，讓程式變成自動循環燈號：

  • 挑戰一：時間固定 (綠2秒 ➜ 黃1秒 ➜ 紅3秒)
  • 挑戰二：時間可調
💡 提示：使用「副程式」&「自定變量」
3
蜂鳴器發聲

接線：蜂鳴器

🎯 任務：單獨接上蜂鳴器，用積木發出聲音。嘗試調整「等待時間」，做出急促的滴滴聲。

💡 提示：因為 ESP32 同時間只能有一個硬體作用，用蜂鳴器播放聲音的時間來取代原有的「等待時間」

程式邏輯挑戰 (話流程圖)

「先亮綠燈 2 秒，此時蜂鳴器慢慢發出『滴—滴』的聲音。接著換黃燈亮，安靜等待 1 秒。最後換紅燈亮 3 秒，此時蜂鳴器變成急促的『滴滴滴滴』。然後不斷重複。」

綠燈亮起 + 蜂鳴器 慢速重複滴滴聲 (2 秒)
黃燈亮起 (等待 1 秒)
紅燈亮起 + 蜂鳴器 急促重複滴滴聲 (3 秒)
虛擬碼 (Pseudo Code) Day 1 
Loop {
  // 1. 綠燈階段 (慢速提示音)
  set_light("GREEN");
  repeat(2) {
    buzzer_beep();
    delay(1000);
  }

  // 2. 黃燈階段 (安靜等待)
  set_light("YELLOW");
  buzzer_stop();
  delay(1000);

  // 3. 紅燈階段 (急促警告音)
  set_light("RED");
  repeat(15) {
    buzzer_beep();
    delay(200);
  }
}

三小時實作通關指南

1
第 1 小時：破冰與神奇開關

🎯 任務目標： 完成任務 0 (測試內建燈閃爍) 與任務 1 (神奇開關)。

  • 接線任務會比較繁瑣，需依照 Pin 腳對應表接上 3 顆按鈕與 3 顆 LED。
  • 寫出 3 組「如果...否則...」的獨立判斷。
老師檢核點： 提醒學生按鈕積木必須選擇「上拉電阻」，否則訊號會浮接亂跳！並引導學生檢查腳位數字是否對應正確。
2
第 2 小時：時間魔法師與聲音

🎯 任務目標： 拿掉按鈕控制，完成任務 2 (自動紅綠燈) 與任務 3 (蜂鳴器發聲)。

  • 利用「副程式」整理綠、黃、紅燈的積木，並挑戰使用「自定變量」來靈活控制秒數。
  • 單獨測試蜂鳴器，體驗利用「蜂鳴器播放時間」來取代原有的「等待」積木。
3
第 3 小時：專題挑戰 —— 有聲音的紅綠燈

🎯 任務目標： 將紅綠燈序列與蜂鳴器提示音完美結合。

  • 依照「話流程圖」中的設計，綠燈時加入慢速蜂鳴器，紅燈時加入急促蜂鳴器，黃燈則保持安靜。
  • 加分挑戰： 加上行人過馬路的按鈕！平常恆亮綠燈，當有人按下按鈕時，才開始執行切換紅燈的保護程序。
