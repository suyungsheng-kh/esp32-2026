Day N：雲端資料庫 —— 數據上雲端

🎯 專題目標：將環境數據即時記錄至 Google 試算表

學會了讀取感測器數值，下一步就是讓這些資料產生長期的價值。今天我們將解鎖 ESP32 的上網功能，將 DHT11 的溫濕度資料，透過 WiFi 即時傳送到 Google 試算表中，打造真正的「物聯網」數據中心！

模組原理與概念總結

WiFi 連線

ESP32 內建強大的 WiFi 模組。在程式一開始，我們必須告訴它無線網路的名稱 (SSID) 與密碼，讓它連上網際網路，這是資料上傳的第一步。

Google Apps Script

試算表不會自己接收資料。我們需要撰寫一段簡單的「雲端小程式 (Apps Script)」，就像一個郵差，負責接收 ESP32 送來的包裹，並填入表格中。

HTTP GET 請求

就像在瀏覽器輸入網址一樣。我們將溫濕度資料打包成網址的一部分 (例如: ?temp=25&humi=60)，傳送給 Google Apps Script 執行寫入。

雲端設定三步曲

1
建立試算表

🎯 任務：

  • 開啟 Google Sheets 建立空白試算表。
  • 在第一列 (A1, B1, C1) 依序輸入標題：「時間戳記」、「溫度」、「濕度」。
2
部署 Apps Script

🎯 任務：

  • 點選「擴充功能 ➜ Apps Script」。
  • 貼上老師提供的專用程式碼並存檔。
  • 部署為「網路應用程式」(權限設為所有人)，並取得 Web App URL。
3
積木設定與測試

🎯 任務：

  • 在 motoBlockly 設定 WiFi 帳密。
  • 將取得的 Web App URL 貼入積木。
  • 組裝讀取 DHT11 (Pin 33) 數值並上傳的積木邏輯。

Google Apps Script 雲端程式碼

這是一段運行在 Google 伺服器上的程式，負責接收 ESP32 送來的 GET 請求，並將裡面的 temp 和 humi 參數寫入試算表的新一列。

function doGet(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  
  // 讀取 URL 中的參數 (?temp=XX&humi=XX)
  var temp = e.parameter.temp || "N/A";
  var humi = e.parameter.humi || "N/A";
  
  // 取得台灣時區的當前時間
  var tz = "Asia/Taipei";
  var now = Utilities.formatDate(new Date(), tz, "yyyy/MM/dd HH:mm:ss");
  
  // 寫入一列資料：[時間, 溫度, 濕度]
  sheet.appendRow([now, temp, humi]);
  
  return ContentService.createTextOutput("Data Received OK");
}

未來發展方向：物聯網的無限可能

三天的基礎課程只是物聯網大門的鑰匙。當你掌握了「將資料傳上網路」的能力後，不妨在未來的專題或科展中，挑戰以下三大進階領域。這些都是目前業界與創客圈最熱門的實作方向：

Data Dashboard
方向一：數據視覺化

打造專業數據儀表板 (Looker Studio)

Google 試算表裡的純文字數據很難一眼看出變化趨勢。透過 Google 的免費工具 Looker Studio (前身為 Data Studio)，你可以無縫匯入試算表資料，不需要寫任何程式碼 (No-code)，只要用滑鼠拖拉，就能生成即時動態的折線圖、圓餅圖與數字儀表板。

基礎起步步驟 (極簡版)：
  1. 登入平台：使用你的 Google 帳號開啟 Looker Studio，點擊「建立空白報告」。
  2. 連接資料：資料來源選擇「Google 試算表」，授權後選取你剛建立的溫濕度紀錄表。
  3. 拉出圖表：從上方選單點選「新增圖表 ➜ 時間序列圖」。將右側設定的Ｘ軸維度設為「時間」，Ｙ軸指標設為「溫度」，即可看見數據折線！
進階挑戰與情境：

試著建置「校園溫室觀測站」，加入溫度「量表 (儀表板)」，並將專屬圖表網址分享給師生觀看。

方向二：雙向互動遙控

手機網頁伺服器 (Web Server)

目前的課程我們是將資料「單向」丟上雲端。真正的物聯網應該要是「雙向」的！你可以將 ESP32 本身變成一台微型的網頁伺服器，當你的手機與 ESP32 連上同一個 WiFi 時，只需在手機瀏覽器輸入 ESP32 的 IP 位置，就能打開專屬的遙控器介面。

基礎起步步驟 (極簡版)：
  1. 積木設定：在 motoBlockly 的通訊分類中，找到「網頁伺服器 (Web Server)」模組積木。
  2. 建立介面：設定伺服器回傳一組最簡單的 HTML 網頁文字，裡面包含兩個超連結按鈕 (例如 /on 開燈 與 /off 關燈)。
  3. 手機連線：啟動 ESP32 並取得它的 IP 位置。手機連上同一個 WiFi，打開瀏覽器輸入該 IP。點擊網頁上的按鈕，ESP32 接收到對應網址就會控制 LED 亮滅！
進階挑戰與情境：

製作「寵物自動餵食器」，在外也能用手機登入網頁，按下按鈕控制馬達轉動掉落飼料；或製作「遠端密碼門鎖」。

方向三：連結真實世界

串接網路開放資料 (Open Data)

做物聯網不代表所有東西都要自己裝感測器！政府機關與各大平台（如氣象署、環境部）提供了許多免費的「開放資料 (API)」。你可以讓 ESP32 主動上網抓取這些數據（例如今日降雨機率、即時空污指數），並結合硬體做出反應。

基礎起步步驟 (極簡版)：
  1. 尋找資料源：尋找免費免註冊的簡單 API (如 World Time API 抓網路時間)，或到氣象署開放資料平台免費註冊取得 API Key。
  2. 發送請求：使用 motoBlockly 的「HTTP GET 請求」積木，將 API 網址填入。ESP32 執行後會從網路抓回一大串包含資料的文字 (通常為 JSON 格式)。
  3. 解析數據：這是一大考驗！使用「JSON 格式解析」積木，精準抓出那一長串文字中的特定數值 (例如今日降雨機率 PoP)。
進階挑戰與情境：

設計一個放在玄關的「智慧出門小精靈」，如果 ESP32 抓到今天該縣市降雨機率大於 50%，出門前經過時就會主動亮起藍燈提醒你要帶傘。
