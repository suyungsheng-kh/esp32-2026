Day N：雲端資料庫 —— 數據上雲端

🎯 專題目標：將環境數據即時記錄至 Google 試算表

學會了讀取感測器數值，下一步就是讓這些資料產生長期的價值。今天我們將解鎖 ESP32 的上網功能，透過建立 Google 表單 作為資料接收的橋樑，將 DHT11 的溫濕度資料，即時傳送到後台的 Google 試算表中，打造真正的「物聯網」數據中心！

全局運作模擬示範

在開始接線與寫程式前，先透過互動模擬器，觀察資料是如何從「感測器」經過「判斷」與「動作」，最終流向「雲端資料庫」的完整路徑。

模組原理與概念總結

WiFi 連線

ESP32 內建強大的 WiFi 模組。在程式一開始，我們必須告訴它無線網路的名稱 (SSID) 與密碼，讓它連上網際網路，這是資料上傳的第一步。

Google 表單 (資料閘門)

試算表不會自己接收外來資料。我們建立一個表單當作「收發室」，當表單收到 ESP32 送來的資料後，就會自動整理並填入試算表中，完全不需寫程式！

HTTP GET 請求

就像在瀏覽器輸入網址一樣。我們利用表單專屬的題目代碼，將溫濕度打包成網址的一部分 (例如: ?entry.123=25)，讓 ESP32 發送出填表請求。

雲端設定三步曲 (Google 表單法)

1
建立表單與試算表

🎯 任務：

  • 新增 Google 表單，建立兩題「簡答題」：溫度、濕度。
  • 點擊上方「回覆」標籤，點選「連結至試算表」，建立一個全新的資料接收表。
2
找出表單代碼 (Entry ID)

🎯 任務：

  • 點擊表單右上角「取得預先填寫的連結」，隨便輸入數字後取得網址。
  • 從網址中找出溫度與濕度對應的 entry.XXXXXX 代號。
  • 將網址中的 viewform 改為 formResponse。
3
積木設定與測試

🎯 任務：

  • 在 motoBlockly 設定 WiFi 帳密。
  • 將修改好的表單網址貼入 HTTP GET 積木。
  • 組裝參數，將 Entry ID 與讀取到的 DHT11 數值連結並發送。

表單網址解析

Google 表單的網址藏著小秘密！只要把「預填網址」稍作修改，ESP32 就能像人類一樣，在背景偷偷幫我們把資料「送出 (Submit)」。

# 步驟 1: 將原本網址最後的 viewform 改為 formResponse
https://docs.google.com/forms/d/e/AKfy.../formResponse
# 步驟 2: 加上問號 (?)，並串接題目的 entry 代碼與數值 (用 & 符號連接)
? entry.123456789=28 & entry.987654321=65

未來發展方向：物聯網的無限可能

三天的基礎課程只是物聯網大門的鑰匙。當你掌握了「將資料傳上網路」的能力後，不妨在未來的專題或科展中，挑戰以下三大進階領域。這些都是目前業界與創客圈最熱門的實作方向：

Data Dashboard
方向一：數據視覺化

打造專業數據儀表板 (Looker Studio)

Google 試算表裡的純文字數據很難一眼看出變化趨勢。透過 Google 的免費工具 Looker Studio (前身為 Data Studio)，你可以無縫匯入試算表資料，不需要寫任何程式碼 (No-code)，只要用滑鼠拖拉，就能生成即時動態的折線圖、圓餅圖與數字儀表板。

基礎起步步驟 (極簡版)：
  1. 登入平台：使用你的 Google 帳號開啟 Looker Studio，點擊「建立空白報告」。
  2. 連接資料：資料來源選擇「Google 試算表」，授權後選取你剛建立的溫濕度紀錄表。
  3. 拉出圖表：從上方選單點選「新增圖表 ➜ 時間序列圖」。將右側設定的Ｘ軸維度設為「時間」，Ｙ軸指標設為「溫度」，即可看見數據折線！
進階挑戰與情境：

試著建置「校園溫室觀測站」，加入溫度「量表 (儀表板)」，並將專屬圖表網址分享給師生觀看。

方向二：雙向互動遙控

手機網頁伺服器 (Web Server)

目前的課程我們是將資料「單向」丟上雲端。真正的物聯網應該要是「雙向」的！你可以將 ESP32 本身變成一台微型的網頁伺服器，當你的手機與 ESP32 連上同一個 WiFi 時，只需在手機瀏覽器輸入 ESP32 的 IP 位置，就能打開專屬的遙控器介面。

基礎起步步驟 (極簡版)：
  1. 積木設定：在 motoBlockly 的通訊分類中，找到「網頁伺服器 (Web Server)」模組積木。
  2. 建立介面：設定伺服器回傳一組最簡單的 HTML 網頁文字，裡面包含兩個超連結按鈕 (例如 /on 開燈 與 /off 關燈)。
  3. 手機連線：啟動 ESP32 並取得它的 IP 位置。手機連上同一個 WiFi，打開瀏覽器輸入該 IP。點擊網頁上的按鈕，ESP32 接收到對應網址就會控制 LED 亮滅！
進階挑戰與情境：

製作「寵物自動餵食器」，在外也能用手機登入網頁，按下按鈕控制馬達轉動掉落飼料；或製作「遠端密碼門鎖」。

方向三：連結真實世界

串接網路開放資料 (Open Data)

做物聯網不代表所有東西都要自己裝感測器！政府機關與各大平台（如氣象署、環境部）提供了許多免費的「開放資料 (API)」。你可以讓 ESP32 主動上網抓取這些數據（例如今日降雨機率、即時空污指數），並結合硬體做出反應。

基礎起步步驟 (極簡版)：
  1. 尋找資料源：尋找免費免註冊的簡單 API (如 World Time API 抓網路時間)，或到氣象署開放資料平台免費註冊取得 API Key。
  2. 發送請求：使用 motoBlockly 的「HTTP GET 請求」積木，將 API 網址填入。ESP32 執行後會從網路抓回一大串包含資料的文字 (通常為 JSON 格式)。
  3. 解析數據：這是一大考驗！使用「JSON 格式解析」積木，精準抓出那一長串文字中的特定數值 (例如今日降雨機率 PoP)。
進階挑戰與情境：

設計一個放在玄關的「智慧出門小精靈」，如果 ESP32 抓到今天該縣市降雨機率大於 50%，出門前經過時就會主動亮起藍燈提醒你要帶傘。

智慧溫室模擬：從感測器到雲端

不需硬體與網路，直接觀察資料流動的完整路徑

這個模擬把一次「迴圈」拆成五個階段輪流亮起：讀取 → 判斷 → 動作 → 上傳 → 等待。 按下「握住感測器」讓溫度上升，觀察積木在哪一格亮起、風扇何時開始轉、曲線何時越過門檻線。

// DHT11 資料腳接在 GPIO 33
#define DHTPIN 33
DHT dht(DHTPIN, DHT11);

// 只做一次：連上 WiFi
void setup() {
    WiFi.begin("CAMP-2G", "password");
    while (WiFi.status() != WL_CONNECTED) delay(500);
    lcd.print("WiFi OK");
}

void loop() {
    float t = dht.readTemperature();   // 讀溫度
    float h = dht.readHumidity();      // 讀濕度
    if (t > 30) {                    // 判斷門檻
    digitalWrite(RELAY, HIGH);       // 風扇轉
    lcd.print("WARNING! FAN ON");
    } else {
    digitalWrite(RELAY, LOW);        // 風扇停
    lcd.print("STATUS OK");
    }
    sendToGoogleSheet(t, h);           // 上傳雲端
    delay(2000);                       // 等 2 秒
}
